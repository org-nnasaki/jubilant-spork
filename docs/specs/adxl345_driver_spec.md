# ADXL345 ドライバ仕様書

**センサー:** Analog Devices ADXL345 — 3軸加速度センサー
**データシート:** ADXL345 (Rev. 0)
**チップID:** 0xE5 (レジスタ 0x00 DEVID)

---

## 1. レジスタマップ

### 1.1 レジスタ一覧

| レジスタ名 | アドレス | R/W | リセット値 | 説明 |
|---|---|---|---|---|
| DEVID | 0x00 | R | 0xE5 | デバイスID (固定値) |
| THRESH_TAP | 0x1D | R/W | 0x00 | タップ検出閾値 (62.5 mg/LSB) |
| OFSX | 0x1E | R/W | 0x00 | X軸オフセット (15.6 mg/LSB) |
| OFSY | 0x1F | R/W | 0x00 | Y軸オフセット (15.6 mg/LSB) |
| OFSZ | 0x20 | R/W | 0x00 | Z軸オフセット (15.6 mg/LSB) |
| DUR | 0x21 | R/W | 0x00 | タップ継続時間 (625 µs/LSB) |
| Latent | 0x22 | R/W | 0x00 | タップ潜伏時間 (1.25 ms/LSB) |
| Window | 0x23 | R/W | 0x00 | タップウィンドウ (1.25 ms/LSB) |
| THRESH_ACT | 0x24 | R/W | 0x00 | アクティビティ閾値 (62.5 mg/LSB) |
| THRESH_INACT | 0x25 | R/W | 0x00 | 非アクティビティ閾値 (62.5 mg/LSB) |

| TIME_FF | 0x29 | R/W | 0x00 | 自由落下時間 (5 ms/LSB) |
| TAP_AXES | 0x2A | R/W | 0x00 | タップ軸制御 |

| INT_SOURCE | 0x30 | R | 0x02 | 割り込みソース |

### 1.2 主要レジスタのビットフィールド

#### BW_RATE (0x2C)

| bit | 名前 | 説明 |
|---|---|---|
| [4] | LOW_POWER | 1=低消費電力モード有効 |
| [3:0] | Rate | 出力データレート (下表参照) |

| Rate[3:0] | 帯域幅 (Hz) | 出力データレート (Hz) |
|---|---|---|
| 0110 | 6.25 | 12.5 |
| 0111 | 12.5 | 25 |
| 1000 | 25 | 50 |
| 1001 | 50 | 100 |
| 1010 | 100 | 200 (デフォルト) |
| 1011 | 200 | 400 |
| 1100 | 400 | 800 |
| 1101 | 800 | 1600 |
| 1110 | 1600 | 3200 |

#### POWER_CTL (0x2D)

| bit | 名前 | 説明 |
|---|---|---|
| [5] | Link | 1=アクティビティ→非アクティビティを直列リンク |
| [4] | AUTO_SLEEP | 1=非アクティビティ検出時にスリープモードへ自動遷移 |
| [3] | Measure | 1=測定モード、0=スタンバイモード |
| [2] | Sleep | 1=スリープモード |
| [1:0] | Wakeup | スリープ中のウェイクアップ頻度 |

| Wakeup[1:0] | 頻度 (Hz) |
|---|---|
| 00 | 8 |
| 01 | 4 |
| 10 | 2 |
| 11 | 1 |

#### DATA_FORMAT (0x31)

| bit | 名前 | 説明 |
|---|---|---|
| [7] | SELF_TEST | 1=セルフテスト力を印加 |
| [6] | SPI | 1=3線SPIモード、0=4線SPIモード |
| [5] | INT_INVERT | 1=割り込みアクティブLow |
| [3] | FULL_RES | 1=フル解像度モード (4 mg/LSB維持) |
| [2] | Justify | 1=左詰め、0=右詰め (MSB符号拡張) |
| [1:0] | Range | 測定レンジ (下表参照) |

| Range[1:0] | レンジ |
|---|---|
| 00 | ±2 g |
| 01 | ±4 g |
| 10 | ±8 g |
| 11 | ±16 g |

#### INT_ENABLE / INT_MAP / INT_SOURCE (0x2E / 0x2F / 0x30)

| bit | 名前 |
|---|---|
| [7] | DATA_READY |
| [6] | SINGLE_TAP |
| [5] | DOUBLE_TAP |
| [4] | Activity |
| [3] | Inactivity |
| [2] | FREE_FALL |
| [1] | Watermark |
| [0] | Overrun |

- INT_ENABLE: 1=該当割り込み有効
- INT_MAP: 0=INT1ピン、1=INT2ピン
- INT_SOURCE: 1=該当イベント発生中 (読み出しでクリアされるものあり)

#### FIFO_CTL (0x38)

| bit | 名前 | 説明 |
|---|---|---|
| [7:6] | FIFO_MODE | FIFOモード (下表参照) |
| [5] | Trigger | トリガイベントのピン選択 (0=INT1, 1=INT2) |
| [4:0] | Samples | FIFOサンプル数 / ウォーターマーク閾値 |

| FIFO_MODE[1:0] | モード | 説明 |
|---|---|---|
| 00 | Bypass | FIFO無効 |
| 01 | FIFO | 32サンプルまで蓄積、満杯で停止 |
| 10 | Stream | 最新32サンプルを保持、古いデータを上書き |
| 11 | Trigger | トリガ前後のデータを保持 |

#### FIFO_STATUS (0x39)

| bit | 名前 | 説明 |
|---|---|---|
| [7] | FIFO_TRIG | トリガイベント発生 |
| [5:0] | Entries | FIFO内のサンプル数 |

## 2. 通信プロトコル手順

### 2.1 SPI (推奨)

- **対応モード:** SPI Mode 11 (CPOL=1, CPHA=1) — データは立下がりエッジで更新、立上がりエッジでサンプリング
- **最大クロック:** 5 MHz
- **CS:** アクティブLow

#### SPIフレーム構造

最初のバイト:
```
bit7: R/W (1=読み出し, 0=書き込み)
bit6: MB  (1=マルチバイト, 0=シングルバイト)
bit5〜bit0: レジスタアドレス (6ビット)
```

#### SPI書き込み手順 (シングルバイト)
```
[CS=Low] [0 | 0 | アドレス(6bit)] [データバイト] [CS=High]
```

#### SPI書き込み手順 (マルチバイト)
```
[CS=Low] [0 | 1 | アドレス(6bit)] [データバイト0] [データバイト1] ... [CS=High]
```
- アドレスは自動インクリメント

#### SPI読み出し手順 (シングルバイト)
```
[CS=Low] [1 | 0 | アドレス(6bit)] [データバイト] [CS=High]
```

#### SPI読み出し手順 (マルチバイト)
```
[CS=Low] [1 | 1 | アドレス(6bit)] [データバイト0] [データバイト1] ... [CS=High]
```
- アドレスは自動インクリメント
- **加速度データ (0x32〜0x37) の読み出しはマルチバイトリードを推奨**（6バイト一括）

### 2.2 I2C

- **デバイスアドレス:** 0x53 (ALT ADDRESS=GND) または 0x1D (ALT ADDRESS=HIGH)
- **対応速度:** Standard (100 kHz), Fast (400 kHz)
- **注意:** I2Cモード使用時はCSピンをVDD_IOに接続すること

#### I2C書き込み手順
```
[S] [スレーブアドレス + W(0)] [ACK] [レジスタアドレス] [ACK] [データ] [ACK] ... [P]
```

#### I2C読み出し手順
```
[S] [スレーブアドレス + W(0)] [ACK] [レジスタアドレス] [ACK]
[Sr] [スレーブアドレス + R(1)] [ACK] [データ0] [ACK] ... [データN] [NACK] [P]
```
- レジスタアドレスは自動インクリメント

### 2.3 初期化手順

1. デバイスIDの確認: レジスタ0x00を読み出し、0xE5であることを確認
2. DATA_FORMAT (0x31) を設定: レンジ、フル解像度モード
3. BW_RATE (0x2C) を設定: 出力データレート
4. INT_ENABLE (0x2E) を設定: 使用する割り込みを有効化
5. FIFO_CTL (0x38) を設定 (FIFO使用時)
6. オフセットキャリブレーション (0x1E〜0x20) を設定 (必要に応じて)
7. POWER_CTL (0x2D) のMeasureビットを1に設定して測定開始

### 2.4 測定データ読み出し手順

1. INT_SOURCE (0x30) のDATA_READYビットを確認 (ポーリング時)、または割り込み待機
2. 0x32〜0x37の6バイトをマルチバイトリード
3. データを結合:
   - x_raw = (DATAX1 << 8) | DATAX0  (signed 16-bit, 有効ビット数はモードによる)
   - y_raw = (DATAY1 << 8) | DATAY0
   - z_raw = (DATAZ1 << 8) | DATAZ0
4. g値に変換（セクション3参照）

## 3. データ変換 (LSB → g)

### 3.1 概要

ADXL345はBoschセンサーのような補正アルゴリズムは不要。生データにスケールファクタを乗じるだけでg値に変換できる。

### 3.2 フル解像度モード (FULL_RES=1)

**すべてのレンジで4 mg/LSBを維持。** レンジを広げると有効ビット数が増える。

| レンジ | 有効ビット数 | スケールファクタ | データ範囲 |
|---|---|---|---|
| ±2 g | 10 bit | 3.9 mg/LSB ≈ 4 mg/LSB | -512 〜 +511 |
| ±4 g | 11 bit | 3.9 mg/LSB ≈ 4 mg/LSB | -1024 〜 +1023 |
| ±8 g | 12 bit | 3.9 mg/LSB ≈ 4 mg/LSB | -2048 〜 +2047 |
| ±16 g | 13 bit | 3.9 mg/LSB ≈ 4 mg/LSB | -4096 〜 +4095 |

```
加速度 [g] = raw_data × 0.004   (= 4 mg/LSB)
加速度 [mg] = raw_data × 4
```

### 3.3 10ビット固定モード (FULL_RES=0)

**すべてのレンジで10ビット固定。** レンジを広げるとスケールファクタが変わる。

| レンジ | 有効ビット数 | スケールファクタ |
|---|---|---|
| ±2 g | 10 bit | 3.9 mg/LSB |
| ±4 g | 10 bit | 7.8 mg/LSB |
| ±8 g | 10 bit | 15.6 mg/LSB |
| ±16 g | 10 bit | 31.2 mg/LSB |

### 3.4 データフォーマット

- **2の補数**: 符号付き整数
- **バイト順**: LSBファースト (DATAx0=下位バイト, DATAx1=上位バイト)
- **右詰め (Justify=0)**: MSBが符号ビット、符号拡張あり → そのままsigned int16_tにキャスト可能
- **左詰め (Justify=1)**: MSBが符号ビット、左詰め → 使用非推奨

### 3.5 オフセットキャリブレーション

OFSX/OFSY/OFSZレジスタ (0x1E〜0x20) に値を書き込むことで、出力値を補正できる。
- スケール: 15.6 mg/LSB
- 例: 1g=0.0 にするためのオフセット値を実測から算出

## 4. 電源モード・動作モード

### 4.1 電源モード一覧

| モード | 設定 | 消費電流 | 説明 |
|---|---|---|---|
| スタンバイ | Measure=0 | 0.1 µA | 測定停止、レジスタアクセス可能 |
| 測定 | Measure=1 | 23〜140 µA (レートによる) | 連続測定 |
| スリープ | Measure=1, Sleep=1 | 低減 | 低頻度のウェイクアップ測定 |
| オートスリープ | AUTO_SLEEP=1, Link=1 + 非アクティビティ閾値設定 | 動的 | アクティビティ検出で自動復帰 |

### 4.2 モード遷移手順

#### スタンバイ → 測定
```
POWER_CTL (0x2D) のbit3 (Measure) を1に設定
```

#### 測定 → スタンバイ
```
POWER_CTL (0x2D) のbit3 (Measure) を0に設定
```

#### スリープモード設定
```
POWER_CTL (0x2D):
  Measure=1, Sleep=1, Wakeup[1:0]=希望頻度
```
- ウェイクアップ頻度: 8Hz/4Hz/2Hz/1Hz

#### オートスリープ設定
```
1. THRESH_INACT (0x25) に非アクティビティ閾値を設定
2. TIME_INACT (0x26) に非アクティビティ時間を設定
3. ACT_INACT_CTL (0x27) で軸有効化
4. THRESH_ACT (0x24) にアクティビティ閾値を設定
5. POWER_CTL (0x2D): Link=1, AUTO_SLEEP=1, Measure=1
```

### 4.3 低消費電力モード

BW_RATE (0x2C) のLOW_POWERビットを1にすると、ノイズがやや増えるが消費電流が低減。

| データレート | 通常電流 | 低電力電流 |
|---|---|---|
| 12.5 Hz | 34 µA | 23 µA |
| 25 Hz | 40 µA | 23 µA |
| 50 Hz | 45 µA | 23 µA |
| 100 Hz | 55 µA | 23 µA |
| 200 Hz | 80 µA | 34 µA |
| 400 Hz | 140 µA | 40 µA |

### 4.4 FIFOモード

| モード | FIFO_MODE[1:0] | 説明 |
|---|---|---|
| Bypass | 00 | FIFO無効、最新1サンプルのみ |
| FIFO | 01 | 32サンプルまで蓄積、満杯で停止 |
| Stream | 10 | 32サンプルのリングバッファ、古いデータを上書き |
| Trigger | 11 | トリガイベント前後のデータを保持 |

- FIFOは32レベル（各レベル=X,Y,Zの3軸分）
- Watermark割り込みでバッチ読み出しが可能
- FIFO内のサンプル数はFIFO_STATUS (0x39) のEntries[5:0]で確認

#### FIFO読み出し手順
1. FIFO_STATUS (0x39) でEntries数を確認
2. DATAX0 (0x32) から6バイトのマルチバイトリードを繰り返す
3. 読み出しごとにFIFOポインタが進む
